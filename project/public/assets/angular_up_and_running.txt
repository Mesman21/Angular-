Частина I: Основи фреймворку та CLI

Розділ 1: Вступ до Angular.
Angular — це потужний фреймворк для створення Single Page Applications (SPA), розроблений Google. Він базується на TypeScript і пропонує повний набір інструментів та патернів для побудови масштабованих додатків. Angular відомий своєю структурою, яка забезпечує високу підтримуваність коду.

Розділ 2: Angular CLI.
Angular Command Line Interface (CLI) є ключем до продуктивності. Замість ручного налаштування модулів, компонентів та конфігурації, CLI дозволяє генерувати boilerplate-код, запускати тести та збирати додаток однією командою. Основні команди: ng new <project-name>, ng generate component <name>, ng serve. CLI також керує конфігурацією Webpack/Vite.

Розділ 3: Компоненти та шаблони.
Компоненти — це основні будівельні блоки Angular. Кожен компонент керує частиною екрана і складається з трьох елементів: TypeScript-класу (логіка), HTML-шаблону (подання) та CSS-стилів. Взаємодія між компонентами відбувається через декоратори @Input() та @Output().

Частина II: Управління даними та RxJS

Розділ 4: Двостороння прив'язка даних.
Angular спрощує синхронізацію даних між компонентами та шаблоном за допомогою двосторонньої прив'язки [(ngModel)]. Це особливо корисно у простих формах. Для складніших сценаріїв рекомендується використовувати реактивний підхід.

Розділ 5: Сервіси та Ін'єкція залежностей (DI).
Сервіси використовуються для відокремлення бізнес-логіки, логіки HTTP-запитів та управління станом від компонентів. DI в Angular — це потужний патерн, який дозволяє компонентам отримувати необхідні залежності (наприклад, HttpClient або BookService) без їх створення, що робить код більш тестованим.

Розділ 6: Робота з RxJS та Observables.
Angular широко використовує бібліотеку Reactive Extensions for JavaScript (RxJS) для обробки асинхронних подій. Observables (Спостережувані) — це основа для обробки потоків даних (HTTP-відповіді, події маршрутизатора, користувацькі події). Використання операторів RxJS (map, filter, switchMap) дозволяє ефективно керувати асинхронним кодом.

Частина III: Маршрутизація та форми

Розділ 7: Angular Router.
Маршрутизація відповідає за навігацію у SPA. Вона дозволяє оголошувати шляхи (/library, /read/:id), захищати маршрути за допомогою Guards (CanActivate) та ліниво завантажувати модулі (Lazy Loading) для оптимізації продуктивності.

Розділ 8: Reactive Forms.
Reactive Forms — це реактивний підхід до управління станом форм. Він дозволяє створювати форми програмно (у класі компонента) і надає повний контроль над валідацією, станом та подіями форми, що є найкращою практикою для складних додатків.
